## Completion Object

The entity that must change is the Codex-DR first proof-run runway.

The completed state is a runway that advances toward a real boxed Codex
recursive deep-research loop against benchmark-facing cases. The next concrete
target is a tiny DRACO smoke path with boxed runner capability, case manifest,

Provider-off files and validators are containment/evidence infrastructure only.
They are not the completed system.

## Intended System

Codex-DR exists to run a real boxed Codex recursive deep-research loop with
adequacy backpressure against benchmark-facing cases.

The intended loop has:

- planner context gathering and plan/adequacy criteria;
- branch agents or boxed branch work;
- pointer / analysis / evidence returns;
- synthesis over admitted evidence and unresolveds;
- reviewer or fact-checker pressure;
- re-entry when review exposes thin evidence or unsupported claims;
- report generation;
- benchmark-facing evaluation and claim boundaries.

Success is not another deterministic fixture. Success is either a real boxed
smoke run or a precise, repo-local proof of the external blocker preventing it.

## Kernel Invariants

- No unattended provider/model-backed run starts without a run-control receipt,
  operational bounds, stop rules, transcript policy, kill path, and
  allowed/non-claim boundaries.
- No benchmark execution starts before dataset access, case manifest, scorer
  policy, and custody path are explicit.
- No raw private, paid, or large benchmark corpus is committed.
- Generated run outputs stay under ignored `sandbox/codex-dr/runs/`,
  `sandbox/codex-dr/tmp/`, or `sandbox/codex-dr/.agent-workspaces/`.
- The root runtime, product service code, Program 80/90 generated proofs, env
  files, and legacy NLSpecs remain out of scope.
- Public Grep/Parcha scores calibrate the target; they do not prove
  Alexandria performance.
- Provider-off fixtures remain non-claims.
